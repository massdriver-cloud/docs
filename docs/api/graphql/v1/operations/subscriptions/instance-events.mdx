Subscribe to events for a specific instance via WebSocket.

Receives events when the instance's configuration changes, a deployment is created,
or an incoming connection is added or removed. Requires read access to the
instance's project.

```graphql
subscription {
  instanceEvents(organizationId: "my-org", instanceId: "my-database") {
    action
    timestamp
    ... on InstanceEvent {
      instance { id name status }
    }
    ... on ConnectionEvent {
      connection { id fromField toField }
    }
  }
}
```


```graphql
instanceEvents(
  organizationId: ID!
  instanceId: ID!
): Event
```

### Type

#### [<code style={{ fontWeight: 'normal' }}><b>Event</b></code>](/api/graphql/v1/types/interfaces/event.mdx) <Badge class="badge badge--secondary " text="interface"/> 
Base interface implemented by all lifecycle events.

Every event carries an `action` describing what happened and a `timestamp` recording
when it occurred. Concrete event types add a field for the affected resource.

Use inline fragments to access the resource-specific fields:

```graphql
subscription {
  projectEvents(organizationId: "my-org", projectId: "my-project") {
    action
    timestamp
    ... on ComponentEvent {
      component { id name }
    }
    ... on LinkEvent {
      link { id fromField toField }
    }
  }
}
```
