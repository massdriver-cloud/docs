A lifecycle event for a realized connection between two instances in an environment.

A connection is the runtime materialization of a blueprint link — it wires one
instance's output artifact into another instance's input field within a single
environment. Emitted when a connection is established or torn down. Subscribe via
`environmentEvents` for every connection in an environment, or via `instanceEvents`
for connections that terminate on a specific instance.


```graphql
type ConnectionEvent implements Event {
  action: EventAction!
  timestamp: DateTime!
  connection: Connection!
}
```

#### [<code style={{ fontWeight: 'normal' }}><b>Event</b></code>](/api/graphql/v1/types/interfaces/event.mdx) <Badge class="badge badge--secondary " text="interface"/> 
Base interface implemented by all lifecycle events.

Every event carries an `action` describing what happened and a `timestamp` recording
when it occurred. Concrete event types add a field for the affected resource.

Use inline fragments to access the resource-specific fields:

```graphql
subscription {
  projectEvents(organizationId: "my-org", projectId: "my-project") {
    action
    timestamp
    ... on ComponentEvent {
      component { id name }
    }
    ... on LinkEvent {
      link { id fromField toField }
    }
  }
}
```
