A bundle placed in a project's blueprint, representing a slot for deployable infrastructure.

A component is the &#x002A;&#x002A;design-time&#x002A;&#x002A; building block of your architecture. It says
"I want a database here" or "I need a Kubernetes cluster there." The component
defines &#x002A;what&#x002A; to deploy; the actual running infrastructure lives in &#x002A;&#x002A;instances&#x002A;&#x002A;
-- one per environment the component is deployed to.

Components are connected to each other via &#x002A;&#x002A;links&#x002A;&#x002A;, which declare that one
component's output (e.g., a connection string) should be wired into another
component's input.


```graphql
type Component {
  id: ID!
  name: String!
  description: String
  tags: Map!
  position: ComponentPosition
  createdAt: DateTime!
  updatedAt: DateTime!
  ociRepo: OciRepo
  project: Project
}
```
