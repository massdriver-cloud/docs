The resource level in the hierarchy where a tag constraint applies.

Tags are set at the specified scope and cascade downward. For example, a tag set at
the `PROJECT` scope is inherited by all environments and instances in that project.


```graphql
enum TagConstraintScope {
  ORGANIZATION
  PROJECT
  ENVIRONMENT
  COMPONENT
  INSTANCE
}
```




### Values

#### [<code style={{ fontWeight: 'normal' }}>TagConstraintScope.<b>ORGANIZATION</b></code>](#organization) \{#organization\} 
Tag is set on the organization itself.


#### [<code style={{ fontWeight: 'normal' }}>TagConstraintScope.<b>PROJECT</b></code>](#project) \{#project\} 
Tag is set on individual projects.


#### [<code style={{ fontWeight: 'normal' }}>TagConstraintScope.<b>ENVIRONMENT</b></code>](#environment) \{#environment\} 
Tag is set on individual environments.


#### [<code style={{ fontWeight: 'normal' }}>TagConstraintScope.<b>COMPONENT</b></code>](#component) \{#component\} 
Tag is set on blueprint components (design-time).


#### [<code style={{ fontWeight: 'normal' }}>TagConstraintScope.<b>INSTANCE</b></code>](#instance) \{#instance\} 
Tag is set on deployed instances (run-time).
