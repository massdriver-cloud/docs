The resource level where a custom attribute applies.

Attributes set at hierarchy scopes (`PROJECT`, `ENVIRONMENT`, `COMPONENT`) cascade
downward — an attribute set at `PROJECT` is inherited by all environments, instances,
deployments, and resources in that project. `REPO`-scoped attributes apply to OCI
repositories; the bundle name is propagated to components, instances, deployments,
and resources via the `md-repo` system attribute.


```graphql
enum AttributeScope {
  PROJECT
  ENVIRONMENT
  COMPONENT
  REPO
}
```




### Values

#### [<code style={{ fontWeight: 'normal' }}>AttributeScope.<b>PROJECT</b></code>](#project) \{#project\} 
Attribute is set on individual projects.


#### [<code style={{ fontWeight: 'normal' }}>AttributeScope.<b>ENVIRONMENT</b></code>](#environment) \{#environment\} 
Attribute is set on individual environments.


#### [<code style={{ fontWeight: 'normal' }}>AttributeScope.<b>COMPONENT</b></code>](#component) \{#component\} 
Attribute is set on blueprint components (design-time).


#### [<code style={{ fontWeight: 'normal' }}>AttributeScope.<b>REPO</b></code>](#repo) \{#repo\} 
Attribute is set on individual OCI repositories (bundles).
