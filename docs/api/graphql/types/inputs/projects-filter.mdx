Filters for narrowing the projects list. All filters are optional and combine with AND logic.


```graphql
input ProjectsFilter {
  name: StringFilter
  search: String
  attributes: [AttributeFilter!]
}
```




### Fields

#### [<code style={{ fontWeight: 'normal' }}>ProjectsFilter.<b>name</b></code>](#name)<Bullet />[<code style={{ fontWeight: 'normal' }}><b>StringFilter</b></code>](/api/graphql/types/inputs/string-filter.mdx) <Badge class="badge badge--secondary " text="input"/> \{#name\} 
Match by the project's exact display name (`eq`) or a set of names (`in`). For partial or approximate matching, use `search` instead.


#### [<code style={{ fontWeight: 'normal' }}>ProjectsFilter.<b>search</b></code>](#search)<Bullet />[<code style={{ fontWeight: 'normal' }}><b>String</b></code>](/api/graphql/types/scalars/string.mdx) <Badge class="badge badge--secondary " text="scalar"/> \{#search\} 
Free-text search across the project's name and description. Matches whole words anywhere in the text, so it is forgiving of partial or out-of-order terms. When `search` is active and no explicit `sort` is provided, results are ranked by relevance.


#### [<code style={{ fontWeight: 'normal' }}>ProjectsFilter.<b>attributes</b></code>](#attributes)<Bullet />[<code style={{ fontWeight: 'normal' }}><b>[AttributeFilter!]</b></code>](/api/graphql/types/inputs/attribute-filter.mdx) <Badge class="badge badge--secondary " text="list"/> <Badge class="badge badge--secondary " text="input"/> \{#attributes\}
