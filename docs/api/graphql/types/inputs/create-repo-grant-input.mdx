Share an OCI repo with recipient projects matching attribute conditions. The caller must have `repo:grant` on the source repo.


```graphql
input CreateRepoGrantInput {
  action: String!
  recipientConditions: Conditions!
}
```




### Fields

#### [<code style={{ fontWeight: 'normal' }}>CreateRepoGrantInput.<b>action</b></code>](#action)<Bullet />[<code style={{ fontWeight: 'normal' }}><b>String!</b></code>](/api/graphql/types/scalars/string.mdx) <Badge class="badge badge--secondary badge--non_null" text="non-null"/> <Badge class="badge badge--secondary " text="scalar"/> \{#action\} 
The action being granted on the repo. Currently the only grantable repo action is `repo:pull` — repo visibility is inferred from any granted action, and publishing is not a sharing concern.


#### [<code style={{ fontWeight: 'normal' }}>CreateRepoGrantInput.<b>recipientConditions</b></code>](#recipient-conditions)<Bullet />[<code style={{ fontWeight: 'normal' }}><b>Conditions!</b></code>](/api/graphql/types/scalars/conditions.mdx) <Badge class="badge badge--secondary badge--non_null" text="non-null"/> <Badge class="badge badge--secondary " text="scalar"/> \{#recipient-conditions\} 
Restrict this grant to recipient projects whose attributes match every condition. Send the literal `"*"` to apply to every project in the organization. Per-key, send `"*"` to match any value or a non-empty list of strings to match a closed set.
