Reset a package to its initialized state. This is useful for:
- Forcing a stuck bundle off the canvas
- Redeploying a package from scratch
- Clearing state when destroy&#x005F;state is true

The package's status will be set to INITIALIZED.

By default, the state will not be destroyed. If you want to destroy the state, set destroy&#x005F;state to true.
By default, the params will not be cleared. If you want to clear the params, set clear&#x005F;params to true.

This mutation is an escape hatch and can lose data. Please make sure to pull any state first.

[Read our manage state guide](https://docs.massdriver.cloud/guides/managing-state#3-pull-existing-state) on setting up your state backend.
