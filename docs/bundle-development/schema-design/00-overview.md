Design parameter schemas for your bundles using JSON Schema (draft-07). This section covers validation patterns and Massdriver-specific extensions.

## In this section

- **[Validation Patterns](./validation-patterns)** - Common JSON Schema patterns for enums, conditionals, ranges, and more
- **[Massdriver Annotations](./massdriver-annotations)** - `$md.*` extensions for immutability, dynamic enums, and sensitive fields

## Quick reference

All schemas are written in YAML format (used in `massdriver.yaml` files) and follow JSON Schema draft-07.
